# MCNAV 全屏 Apple + Minecraft 视觉细化设计

日期：2026-07-02

## 1. 背景与目标

当前 MCNAV 页面已经完成 React + Markdown 静态导航站基础实现，但视觉上仍存在两个问题：

1. 页面主体被限制在中间最大宽度，宽屏下左右留空明显，用户希望页面视觉铺满整个浏览器。
2. 当前绿色偏纯，Minecraft 氛围不够精致；用户明确希望是 Apple 风格 UI，同时带 Minecraft 像素/方块感。

本次细化目标是在不改变数据架构、不增加后台功能的前提下，重做视觉层：页面全屏铺满，采用 Apple 主导的玻璃、留白、柔和阴影和高级灰绿背景，并用低饱和矿物绿渐变与少量像素方块纹理强化 MCNAV 的 Minecraft 识别。

## 2. 视觉方向

采用用户确认的方向：**Apple 主导，Minecraft 像素点缀**。

核心原则：

- 页面铺满整个浏览器宽度和高度，不再像一个居中的窄容器。
- Apple 风格占主导：大面积留白、玻璃面板、柔和阴影、清晰层级、通透背景。
- Minecraft 风格作为点缀：方块网格、像素块装饰、草方块语义、品牌绿色渐变。
- 不使用刺眼纯绿，不做重像素风，不让页面变成低龄卡通游戏站。

## 3. 白天模式配色

白天模式使用暖白玻璃与低饱和矿物绿渐变。

建议色板：

- 背景暖白：`#FBFCFA`
- 暖白玻璃：`#EEF4EF`
- 苔藓绿：`#6F8F68`
- 柔和草绿：`#9DBB83`
- 深云杉绿：`#4F7658`
- 主文本深绿黑：`#1F2D22`
- 次级文本灰绿：`#657264`

视觉用法：

- 背景使用 `#FBFCFA -> #EEF4EF -> #DBE9DF` 的柔和渐变。
- 品牌块、重点按钮、选中态使用苔藓绿到柔和草绿的渐变。
- 像素块装饰使用透明度较低的苔藓绿/深云杉绿，避免抢正文注意力。
- 面板保持半透明暖白玻璃效果，边框使用低透明灰绿。

## 4. 夜间模式配色

夜间模式使用深云杉黑与墨绿玻璃，不使用死黑背景或荧光绿。

建议色板：

- 深云杉黑：`#071109`
- 墨绿玻璃：`#122015`
- 深卡片面：`#152418`
- 低饱和翡翠：`#8CAD78`
- 夜间正文：`#F4FAF2`
- 夜间次级文字：`#AEBDAE`

视觉用法：

- 背景使用 `#071109 -> #101A12 -> #182718` 深色渐变。
- 绿色只用于品牌块、光晕、选中态和少量像素块，不做大面积荧光铺色。
- 面板使用深墨绿半透明玻璃，边框使用低透明浅绿。
- Hero 区可加入低亮度径向光晕，让夜间模式有深度但不刺眼。

## 5. 布局调整

### 5.1 桌面端

- `.app-shell` 从当前 `width: min(1440px, 100%)` 改为全宽布局。
- 页面使用 `min-height: 100vh`，背景铺满整个视口。
- 保留左侧粘性分类栏，但让它与主内容一起构成全屏仪表盘布局。
- 左右外边距只保留安全 padding，例如 `clamp(16px, 2vw, 32px)`，不再限制整体最大宽度。
- 主内容区可以有最大可读宽度控制在内部元素上，但背景、网格和整体空间必须铺满。

### 5.2 平板和移动端

- 平板端侧边栏切换为顶部/横向分类导航，保持全宽。
- 移动端继续使用单列卡片，但背景和面板仍铺满屏幕宽度。
- 搜索栏与 Hero 在移动端保持充足留白，避免像素装饰压缩内容。

## 6. 组件视觉细化

### Sidebar

- 保持玻璃侧栏，但增加更柔和的 Apple 风边框和阴影。
- 品牌 `M` 使用矿物绿渐变，不再使用纯绿。
- 当前/hover 分类项使用浅绿渐变底，不使用高饱和实色。

### Hero

- Hero 是主要视觉中心：大标题、暖白玻璃面板、右下角低透明像素块。
- 背景增加柔和径向绿光，但透明度要低。
- 方块纹理只做装饰，不覆盖文字区域。

### SearchPanel

- 搜索框使用玻璃胶囊形，阴影更轻。
- 搜索区域铺满主内容宽度，避免孤立在中间。

### NavCard

- 卡片保持 Apple 式圆角和玻璃质感。
- Hover 时使用轻微上浮和柔和绿色边框/阴影。
- 不把卡片做成硬像素块；Minecraft 感主要来自图标容器、背景纹理和品牌色。

### Pixel Accent

- 新增统一像素装饰样式，建议只出现在：背景、Hero 角落、部分品牌区域。
- 像素块需要低透明度和渐变遮罩，避免影响阅读。

## 7. 实现边界

本次只改视觉和布局：

- 修改 `src/App.css` 为主。
- 必要时给 `src/App.tsx` 添加少量装饰容器或 className。
- 不修改 Markdown 数据结构。
- 不修改搜索逻辑、主题逻辑或 favicon fallback 逻辑。
- 不引入 UI 组件库或动画库。
- 不修改 `wwwroot/`。

## 8. 验证标准

完成后应满足：

1. 宽屏下页面背景、布局和视觉元素铺满整个浏览器，不再只有中间区域像页面。
2. 白天模式是暖白玻璃 + 柔和矿物绿渐变，不使用刺眼纯绿。
3. 夜间模式是深云杉黑 + 墨绿玻璃 + 低饱和翡翠光感，不使用死黑或荧光绿。
4. Minecraft 像素元素明显但克制，只作为品牌和装饰点缀。
5. 页面仍符合原有功能：分类导航、搜索、主题切换、链接卡片和图标 fallback 不回退。
6. `npm run test -- --run` 通过。
7. `npm run build` 通过。
